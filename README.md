# funGPT

Ask a question and get an answer in character — as Yoda, Winny The Pooh, Hulk or anyone else. Responses are generated by Gemini and streamed token-by-token to the browser using **SSE (Server-Sent Events)**, so text appears as it's produced rather than after the full response is ready.

SSE is a lightweight HTTP protocol where the server holds a connection open and pushes `data:` frames to the client over plain text. It fits LLM streaming well because the data flows in one direction (server → client), requires no WebSocket handshake, and works natively in browsers via the `EventSource` API.

**Live demo:** https://alanansari.github.io/funGPT/

## Stack

- **Client** — React + Vite (GitHub Pages)
- **Server** — Express + `@google/generative-ai`, streaming via SSE

## Setup

```bash
# Server
cd server
cp .env.example .env   # fill in GEMINI_API_KEY, PORT, MAXREQ
npm install
npm run dev

# Client
cd client
npm install
npm run dev
```

## Testing SSE

The single endpoint streams response chunks as `text/event-stream`.

```
GET /customgpt?character=<name>&question=<text>
```

### curl

```bash
curl -N "http://localhost:3000/customgpt?character=Yoda&question=What+is+the+force"
```

`-N` disables buffering so chunks print as they arrive. Each line looks like:

```
data: The Force, an energy field it is...

data: END
```

The server sends `data: END\n\n` to signal completion.

### Browser DevTools

Open the **Network** tab, filter by **EventStream**, and trigger a request. The **EventStream** sub-tab shows each `data:` frame in real time.

### JavaScript (EventSource)

```js
const url = new URL('http://localhost:3000/customgpt');
url.searchParams.set('character', 'Sherlock Holmes');
url.searchParams.set('question', 'How do you solve a case?');

const es = new EventSource(url);

es.onmessage = ({ data }) => {
  if (data === 'END') { es.close(); return; }
  process.stdout.write(data);   // or append to DOM
};

es.onerror = (err) => { console.error(err); es.close(); };
```

### Rate limiting

Requests are capped at `MAXREQ` per IP per 5-minute window (configured in `.env`). Exceeding it returns HTTP 429.
